Campaign Overview

Our incident response teams have been engaged with multiple healthcare organizations responding to LockBit 3.0 infections since late October 2024. The attacks share common characteristics including initial access through compromised VPN appliances and rapid lateral movement through Active Directory environments. Organizations in Chicago, Houston, Toronto, and Phoenix have been particularly impacted.

The threat actors demonstrated deep knowledge of healthcare IT infrastructure, specifically targeting Electronic Health Record (EHR) systems, medical imaging archives (PACS), and laboratory information systems. The encryption routine was designed to maximize operational impact while preserving some systems needed for ransom negotiation.

Attack Chain Analysis

Phase 1: Initial Access

The threat actors gained initial access through exploitation of CVE-2024-21887, a critical vulnerability in Ivanti Connect Secure VPN appliances. Organizations that had not applied the January 2024 security patches were particularly vulnerable. Initial reconnaissance commands were executed to enumerate the network topology and identify high-value targets.

Phase 2: Credential Harvesting

Once inside the network, attackers deployed Mimikatz variants to harvest credentials from memory. The attackers specifically targeted service accounts with elevated privileges, particularly those used for backup operations and database access. Beaconing traffic was observed to infrastructure at 192.168.45.128 and command callbacks to honey.scanme.sh for DNS tunneling.

# Observed PowerShell reconnaissance commands Get-ADComputer -Filter * | Select-Object Name,OperatingSystem Get-ADUser -Filter {ServicePrincipalName -ne "$null"} -Properties ServicePrincipalName Invoke-Command -ComputerName "oaev-test-win-05" -ScriptBlock { Get-Process } nltest /dclist:healthcare.local

Phase 3: Lateral Movement

Lateral movement was achieved through a combination of Remote Desktop Protocol (RDP), Windows Management Instrumentation (WMI), and PsExec. The attackers moved methodically through the network, targeting systems with access to patient data and critical healthcare applications.

Recommendations

Healthcare organizations should implement the following mitigations immediately:

  1. Patch all Ivanti Connect Secure appliances to the latest version
  2. Implement network segmentation between clinical and administrative systems
  3. Enable multi-factor authentication for all remote access
  4. Review and restrict service account privileges
  5. Implement offline backup solutions for critical systems
  6. Deploy endpoint detection and response (EDR) solutions across all servers
